## Import strategies

Choose the right approach based on your use case:

### 1. Named Imports

:::tip[When to use]
Use named imports when you:

- 📦 Work offline or in restricted network environments
- ✅ Know which flags you need at build time
- 🚀 Want synchronous rendering without loading states

:::

**Best for static flags known at build time.**

- **Bundle impact:** ~950 bytes per flag
- **Tree-shaking:** ✅ Excellent
- **Loading:** Synchronous, instant

For guaranteed per-flag isolation across Vite, webpack, and framework SSR bundlers, import each
component from its `flags/*` subpath:

<Tabs syncKey="framework">
  <TabItem label="React" icon="seti:react">

```tsx
import { FlagUs } from '@sankyu/react-circle-flags/flags/us'
import { FlagCn } from '@sankyu/react-circle-flags/flags/cn'
import { FlagGb } from '@sankyu/react-circle-flags/flags/gb'

export function Header() {
  return (
    <div>
      <FlagUs width={32} height={32} />
      <FlagCn width={32} height={32} />
      <FlagGb width={32} height={32} />
    </div>
  )
}
```

  </TabItem>
  <TabItem label="Vue 3" icon="seti:vue">

```vue
<script setup lang="ts">
import { FlagUs } from '@sankyu/vue-circle-flags/flags/us'
import { FlagCn } from '@sankyu/vue-circle-flags/flags/cn'

<template>
  <div>
    <FlagUs :width="32" :height="32" />
    <FlagCn :width="32" :height="32" />
    <FlagGb :width="32" :height="32" />
  </div>
</template>
```

  </TabItem>
  <TabItem label="Solid.js" icon="seti:javascript">


The shorter main-entry form (`import { FlagUs } from '@sankyu/react-circle-flags'`) remains
supported. Use the explicit subpath when bundle size must be deterministic, because the main entry
also exposes `DynamicFlag` and its complete registry.

### 2. DynamicFlag

<span id="3-dynamicflag"></span>

:::tip[When to use]
Use DynamicFlag when you:

- ⚡ Need to render _any_ flag synchronously from runtime ISO codes
- ❌ Cannot access external CDNs (offline, intranet, air-gapped)
- 🛠️ Want full SVG props support (direct `<svg />` rendering)

:::

:::caution[Large Bundle Impact]
This component adds **~600 KB** to your bundle (it bundles ALL 400+ flags).
:::

**Offline-first runtime flags (bundles all flags).**

- **Bundle impact:** ~600 KB
- **Tree-shaking:** ❌ None
- **Loading:** Synchronous, instant

Render any flag from runtime ISO codes without network requests:

<Tabs syncKey="framework">
  <TabItem label="React" icon="seti:react">

```tsx
import { DynamicFlag } from '@sankyu/react-circle-flags'

export function CountryFlag({ code }: { code: string }) {
  return <DynamicFlag code={code} width={36} height={36} />
}
```

  </TabItem>
  <TabItem label="Vue 3" icon="seti:vue">

```vue
<script setup lang="ts">
import { DynamicFlag } from '@sankyu/vue-circle-flags'
const props = defineProps<{ code: string }>()
</script>

<template>
  <DynamicFlag :code="props.code" :width="36" :height="36" />
</template>
```

  </TabItem>
  <TabItem label="Solid.js" icon="seti:javascript">

```tsx
import { DynamicFlag } from '@sankyu/solid-circle-flags'

export function CountryFlag(props: { code: string }) {
  return <DynamicFlag code={props.code} width={36} height={36} />
}
```

  </TabItem>
</Tabs>

:::note[API Reference]
See full props definition: [DynamicFlagProps →](/reference/api/typealiasdynamicflagprops/)
:::

## Naming convention

All flag components follow a consistent naming pattern:

<CardGrid>
  <Card title="Pattern" icon="star">
    Components use the format **`Flag{ISO}`** where ISO is the uppercase country code.
  </Card>
  <Card title="Examples" icon="list-format">
    - `FlagUs` → United States

    - `FlagCn` → China

    - `FlagGb` → United Kingdom

    - `FlagJp` → Japan

  </Card>
</CardGrid>
