### 1. Named Imports

:::tip[When to use]
Use named imports when you:

- ‚úÖ You know which flags you need at build time
- üöÄ Want synchronous rendering without loading states
- üì¶ Working offline or in restricted network environments

:::

**Best for static flags known at build time.**

- **Bundle impact:** ~950 bytes per flag
- **Tree-shaking:** ‚úÖ Excellent
- **Loading:** Synchronous, instant

Import only the flags you need from the main entry:

<Tabs syncKey="framework">
  <TabItem label="React" icon="seti:react">

```tsx
import { FlagUs, FlagCn, FlagGb } from '@sankyu/react-circle-flags'

export function Header() {
  return (
    <div>
      <FlagUs width={32} height={32} />
      <FlagCn width={32} height={32} />
      <FlagGb width={32} height={32} />
    </div>
  )
}
```

  </TabItem>
  <TabItem label="Vue 3" icon="seti:vue">

```vue

import { FlagUs, FlagCn, FlagGb } from '@sankyu/vue-circle-flags'
</script>

<template>
  <div>

    <FlagCn :width="32" :height="32" />

  </div>
</template>
```

  </TabItem>

### 2. CircleFlag

:::tip[When to use]
Use CircleFlag when you:

- ‚úÖ Country codes are determined at runtime (e.g., user input, API responses)
- üìâ Want to minimize bundle size
- üåê Can tolerate async loading with brief loading states
- üîå Have reliable CDN access

:::

**Best for dynamic flags.**

- **Bundle impact:** ~2 KB (component only)
- **Tree-shaking:** ‚úÖ Perfect
- **Loading:** Async from CDN

Load SVG flags from CDN on demand. Perfect for runtime-determined country codes:

<Tabs groupId="framework">
  <TabItem label="React" icon="seti:react">

```tsx
import { CircleFlag } from '@sankyu/react-circle-flags'

export function CountryFlag({ code }: { code: string }) {
  return <CircleFlag countryCode={code} width={36} height={36} />
}
```

  </TabItem>
  <TabItem label="Vue 3" icon="seti:vue">

```vue
<script setup lang="ts">
import { CircleFlag } from '@sankyu/vue-circle-flags'
const props = defineProps<{ code: string }>()
</script>

<template>
  <CircleFlag :countryCode="props.code" :width="36" :height="36" />
</template>
```

  </TabItem>
</Tabs>

:::note[API Reference]
See full props definition: [CircleFlagProps ‚Üí](/reference/api/interfacecircleflagprops/)
:::

### 3. DynamicFlag

:::tip[When to use]
Use DynamicFlag when you:

- ‚ùå Cannot access external CDNs
- ‚ö° Need instant rendering without any loading states
- üõ†Ô∏è Want to render any flag synchronously at runtime

:::

:::caution[Large Bundle Impact]
This component adds **~600 KB** to your bundle.
:::

**Loads all flags in a single bundle.**

- **Bundle impact:** ~600 KB
- **Tree-shaking:** ‚ùå None
- **Loading:** Synchronous, instant

Bundles ALL 400+ flag components. Only use when you absolutely need synchronous rendering of any flag:

<Tabs syncKey="framework">
  <TabItem label="React" icon="seti:react">

```tsx
import { DynamicFlag } from '@sankyu/react-circle-flags'

export function CountryFlag({ code }: { code: string }) {
  return <DynamicFlag code={code} width={36} height={36} />
}
```

</TabItem>
  <TabItem label="Vue 3" icon="seti:vue">

```vue
<script setup lang="ts">
import { DynamicFlag } from '@sankyu/vue-circle-flags'
const props = defineProps<{ code: string }>()
</script>

<template>
  <DynamicFlag :code="props.code" :width="36" :height="36" />
</template>
```

  </TabItem>
</Tabs>

:::note[API Reference]
See full props definition: [DynamicFlagProps ‚Üí](/reference/api/interfacedynamicflagprops/)
:::

## Naming convention

All flag components follow a consistent naming pattern:

<CardGrid>
  <Card title="Pattern" icon="star">
    Components use the format **`Flag{ISO}`** where ISO is the uppercase country code.
  </Card>
  <Card title="Examples" icon="list-format">
    - `FlagUs` ‚Üí United States

    - `FlagCn` ‚Üí China

    - `FlagGb` ‚Üí United Kingdom

    - `FlagJp` ‚Üí Japan

  </Card>
</CardGrid>
